📝 DingDing 프로젝트 상세 분석 보고서

1. 프로젝트 개요

이 프로젝트는 Django와 Django Rest Framework(DRF)를 기반으로 구축된 아이돌 팬덤 플랫폼 API 서버입니다. 사용자 역할(팬, 아이돌, 매니저, 관리자)에 따라 아이돌/그룹 정보 조회, 스케줄 관리, 북마크 등 다양한 기능을 API 형태로 제공합니다.

2. API 명세 요약

프로젝트의 주요 API 엔드포인트와 담당 View는 다음과 같습니다.

HTTP Method & Path 담당 View 이름 설명
POST /api/v1/users/signup/ UserSignupView 회원가입
POST /api/v1/users/login/ UserLoginView 로그인 (JWT 토큰 발급)
GET, PATCH /api/v1/users/profile/ UserProfileView 프로필 조회 및 수정
GET /api/v1/users/mainboard/ FanMainboardView 팬 메인보드 (오늘의 스케줄) 조회
GET /api/v1/idols/ IdolListView 전체 아이돌 목록 조회 (검색 가능)
GET /api/v1/idols/{idol_id}/ IdolDetailView 특정 아이돌 상세 정보 조회
GET, POST /api/v1/schedules/manager/ ManagerScheduleViewSet 매니저가 담당하는 아이돌의 스케줄 조회 및 생성
PUT, DELETE /api/v1/schedules/manager/{schedule_id}/ ManagerScheduleViewSet 매니저가 담당하는 아이돌의 스케줄 수정 및 삭제

4. 핵심 로직 및 코드 분석

4.1. 사용자 정의 모델: CustomUser

Django의 기본 User 모델을 확장하여 서비스에 필요한 필드(nickname, role 등)를 추가하고, email을 주요 식별자로 사용합니다.

apps/users/models.py


# CustomUser 모델 정의
class CustomUser(AbstractUser):
    # Django의 기본 username 필드를 제거하고 email을 사용자 이름으로 사용합니다.
    username = None

    # 이메일 주소 (필수, 고유)
    email = models.EmailField(
        unique=True,
        null=False,
        blank=False,
        verbose_name="이메일 주소",
    )

    # 닉네임 (필수, 고유)
    nickname = models.CharField(
        max_length=20, unique=True, null=False, blank=False, verbose_name="닉네임"
    )

    # 프로필 이미지 ID (Image 모델의 ForeignKey)
    profile_image = models.ForeignKey(
        Image,
        on_delete=models.SET_NULL,  # 이미지가 삭제되어도 사용자 정보는 유지
        null=True,
        blank=True,
        related_name="user_profiles",
    )
    # 사용자 역할 (NORMAL, IDOL, MANAGER, ADMIN)
    ROLE_CHOICES = (
        ("NORMAL", "일반 사용자"),
        ("IDOL", "아이돌"),
        ("MANAGER", "매니저"),
        ("ADMIN", "관리자"),
    )
    role = models.CharField(
        max_length=10, choices=ROLE_CHOICES, default="NORMAL", null=False, blank=False
    )

    # 이메일을 사용자 이름 필드로 설정합니다.
    USERNAME_FIELD = "email"
    # 필수 필드를 정의합니다.
    REQUIRED_FIELDS = ["nickname"]

    # CustomUserManager를 이 모델의 관리자로 설정합니다.
    objects = CustomUserManager()

    def __str__(self):
        return self.email

4.2. 회원가입 로직

담당 View: UserSignupView

회원가입 요청을 받아 사용자를 생성합니다. 데이터 유효성 검증과 실제 생성을 담당하는 UserSignupSerializer를 사용합니다.

apps/users/views.py


class UserSignupView(generics.CreateAPIView):
    queryset = CustomUser.objects.all()
    serializer_class = UserSignupSerializer
    permission_classes = (AllowAny,) # 누구나 접근 가능하도록 설정

    def create(self, request, *args, **kwargs):
        try:
            serializer = self.get_serializer(data=request.data)
            # Serializer의 is_valid를 호출하여 데이터 유효성 검증
            serializer.is_valid(raise_exception=True)
            # Serializer의 save -> create 메서드를 호출하여 사용자 생성
            user = serializer.save()

            return Response(
                {
                    "message": "회원가입이 성공적으로 완료되었습니다.",
                    "user_id": user.id,
                },
                status=status.HTTP_201_CREATED,
            )
        except ValidationError as e:
            # 유효성 검증 실패 시 에러 처리
            return Response(e.detail, status=status.HTTP_400_BAD_REQUEST)

담당 Serializer: UserSignupSerializer

입력받은 데이터(이메일, 닉네임, 비밀번호 등)의 유효성을 검증하고, create 메서드를 통해 실제 CustomUser 객체를 만듭니다.

apps/users/serializers.py


class UserSignupSerializer(serializers.ModelSerializer):
    password = serializers.CharField(write_only=True, required=True)
    password_confirm = serializers.CharField(write_only=True, required=True)

    class Meta:
        model = CustomUser
        fields = ["email", "nickname", "password", "password_confirm", "profile_image"]

    def validate(self, data):
        # 1. 비밀번호와 비밀번호 확인 필드가 일치하는지 검사
        if data["password"] != data["password_confirm"]:
            raise serializers.ValidationError("비밀번호가 일치하지 않습니다.")
        
        # 2. 이메일, 닉네임 중복 검사
        if CustomUser.objects.filter(email=data["email"]).exists():
            raise serializers.ValidationError("이미 사용 중인 이메일 주소입니다.")
        if CustomUser.objects.filter(nickname=data["nickname"]).exists():
            raise serializers.ValidationError("이미 사용 중인 닉네임입니다.")

        return data

    def create(self, validated_data):
        # validated_data에서 profile_image를 분리
        profile_image_file = validated_data.pop("profile_image", None)

        # CustomUserManager의 create_user를 사용해 사용자 생성
        user = CustomUser.objects.create_user(
            email=validated_data["email"],
            password=validated_data["password"],
            nickname=validated_data["nickname"],
        )

        # 프로필 이미지가 있으면 Image 객체를 만들어 연결
        if profile_image_file:
            image_instance = Image.objects.create(image_file=profile_image_file)
            user.profile_image = image_instance
            user.save()
        return user

4.3. 매니저의 스케줄 관리 로직

담당 ViewSet: ManagerScheduleViewSet

매니저가 담당하는 아이돌의 스케줄을 생성, 조회, 수정, 삭제(CRUD)하는 모든 로직을 담당합니다. ViewSet을 사용하여 관련 로직을 하나의 클래스로 묶었습니다.

apps/schedules/views.py


class ManagerScheduleViewSet(viewsets.ModelViewSet):
    serializer_class = IdolScheduleSerializer
    permission_classes = [IsManagerOrAdmin] # 매니저, 관리자만 접근 가능

    def get_queryset(self):
        # 현재 로그인한 매니저가 담당하는 아이돌 목록을 가져옴
        managed_idols = IdolManager.objects.filter(user=self.request.user).values_list('idol', flat=True)
        # 해당 아이돌의 스케줄만 필터링하여 반환
        return IdolSchedule.objects.filter(idol__in=managed_idols)

    def perform_create(self, serializer):
        # 스케줄을 생성하기 전, 권한 확인
        idol_id = self.request.data.get('idol')
        managed_idols = IdolManager.objects.filter(user=self.request.user).values_list('idol__id', flat=True)
        
        # 요청된 idol_id가 매니저가 담당하는 아이돌이 아니면 권한 없음 에러 발생
        if int(idol_id) not in managed_idols:
            raise PermissionDenied("이 아이돌의 스케줄을 생성할 권한이 없습니다.")
        
        # manager 필드를 현재 로그인한 사용자로 설정하여 저장
        serializer.save(manager=self.request.user)

    def perform_update(self, serializer):
        # 수정/삭제 전에도 객체 수준에서 권한을 확인하는 로직이 내장되어 있음
        self.check_object_permissions(self.request, serializer.instance)
        serializer.save()
